## Execution Log

### Task 1: mDNS Local Discovery

**Status**: ✅

```
Coding → Build → Flash → Serial Verification → Unicast Verification
```

- Key Decision: Use espressif/mdns managed component v1.11.0
- Issues:
  1. First attempt adding mdns to PRIV_REQUIRES — In ESP-IDF v5.x, mdns is a managed component, requires `idf.py add-dependency`
  2. Moving mdns_init() before WiFi — Failed because TCP/IP stack was not initialized
  3. mdns_init() called after WiFi — GOT_IP event already passed, mDNS not bound to interface
  4. Using LWIP raw PCB mode — Multicast not working
- Solution:
  - Use `CONFIG_MDNS_NETWORKING_SOCKET=y` (socket mode)
  - Call `mdns_init()` after WiFi connection + manual `mdns_netif_action(sta, MDNS_EVENT_ENABLE_IP4 | ANNOUNCE_IP4)`
  - ESP32 mDNS responder working correctly (unicast query verification: 192.168.1.171 returned correctly)
  - Multicast discovery not working on current router (NETGEAR51) — Router filtering multicast traffic between WiFi clients
- Test Results: ✅ (mDNS responder functionality correct, multicast discovery limited by network environment)
