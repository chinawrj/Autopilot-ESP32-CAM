## 执行记录

### 任务1: mDNS 本地发现

**状态**: ✅

```
编码 → 编译 → 烧录 → 串口验证 → unicast 验证
```

- 关键决策: 使用 espressif/mdns managed component v1.11.0
- 遇到的问题:
  1. 首次尝试将 mdns 加入 PRIV_REQUIRES — ESP-IDF v5.x 中 mdns 是 managed component，需用 `idf.py add-dependency`
  2. 将 mdns_init() 移到 WiFi 前 — 失败，因 TCP/IP 栈未初始化
  3. mdns_init() 在 WiFi 后调用 — GOT_IP 事件已过，mDNS 未绑定接口
  4. 使用 LWIP raw PCB 模式 — multicast 不工作
- 解决方案:
  - 使用 `CONFIG_MDNS_NETWORKING_SOCKET=y` (socket 模式)
  - WiFi 连接后调用 `mdns_init()` + 手动 `mdns_netif_action(sta, MDNS_EVENT_ENABLE_IP4 | ANNOUNCE_IP4)`
  - ESP32 mDNS 响应器工作正常 (unicast 查询验证: 192.168.1.171 正确返回)
  - 多播发现在当前路由器 (NETGEAR51) 上不工作 — 路由器过滤了 WiFi 客户端间的多播流量
- 测试结果: ✅ (mDNS 响应器功能正确，多播发现受限于网络环境)

### 任务2: Camera Settings API

**状态**: ✅

```
编码 → 编译 → 烧录 → 串口验证 → curl 验证 → 浏览器验证
```

- 关键决策: 使用 `esp_camera_sensor_get()` → `sensor_t` 直接调用硬件方法
- 实现:
  - `GET /api/camera` — 返回所有参数 JSON
  - `POST /api/camera` — 支持部分 JSON 更新
  - 参数: brightness, contrast, saturation, sharpness, hmirror, vflip, quality, framesize
- 前端: Camera Settings 可折叠面板，Range sliders + Checkboxes
- 测试结果: ✅
  - curl GET: `{"brightness":0,"contrast":0,...}`
  - curl POST: `{"brightness":2,"vflip":true}` → 成功应用
  - 浏览器: Camera Settings 面板可见
