## Goals

## Completed Work

### 1. Security Fix: Path Traversal Vulnerability

**Problem**: In `sd_handlers.c`, user-supplied paths were directly concatenated to filesystem paths, allowing attackers to access files outside the SD card mount point via
`/api/sd/list?path=../../../etc` or `/api/sd/file/../../../etc/passwd`.

**Fix**:
- Created `main/path_utils.c/h` — path validation utilities
  - `path_is_safe()`: detects `..` traversal, absolute paths, backslashes
  - `path_sanitize_sd()`: safely constructs SD card paths
- Applied in `sd_list_handler`, `sd_file_handler`, `sd_delete_handler`

## Technical Notes

- Path traversal is one of the most common web server security vulnerabilities
- In C, snprintf does not perform path safety checks — manual validation is required
- ESP32 uses VFS; `/sdcard/../` can actually escape the SD card mount point
- During testing, discovered a bug in using `memchr` + `strlen` to detect embedded null bytes
  (C strings are null-terminated by nature) — switched to a simpler blocklist check
