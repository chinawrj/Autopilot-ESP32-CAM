## 目标

修复 SD 卡 HTTP 处理器中的路径遍历漏洞，扩展单元测试覆盖率。

## 完成内容

### 1. 安全修复: 路径遍历漏洞

**问题**: `sd_handlers.c` 中用户提供的路径直接拼接到文件系统路径，攻击者可通过
`/api/sd/list?path=../../../etc` 或 `/api/sd/file/../../../etc/passwd` 访问
SD 卡之外的文件系统。

**修复**:
- 新建 `main/path_utils.c/h` — 路径验证工具
  - `path_is_safe()`: 检测 `..` 遍历、绝对路径、反斜杠
  - `path_sanitize_sd()`: 安全拼接 SD 卡路径
- 在 `sd_list_handler`、`sd_file_handler`、`sd_delete_handler` 中应用

## 技术笔记

- 路径遍历是 Web 服务器最常见的安全漏洞之一
- C 语言中 snprintf 不做路径安全检查，必须手动验证
- ESP32 使用 VFS，`/sdcard/../` 实际可以跳出 SD 卡挂载点
- 测试中发现 `memchr` + `strlen` 检测嵌入 null byte 的方法有 bug（C 字符串本身
  以 null 结尾），改用更简洁的黑名单检查
