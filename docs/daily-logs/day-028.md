## 完成内容

### 1. System Diagnostics API

新增 `GET /api/system/info` 端点，返回设备深度诊断信息：

```json
{
  "chip": { "model": "ESP32", "cores": 2, "revision": 301, "wifi": true, "bt": true },
  "idf_version": "v5.5.1",
  "memory": { "heap_free": 4169068, "heap_min": 4166160, "psram_free": 4037552, "psram_total": 4194304 },
  "uptime_s": 51, "uptime": "0d 00:00:51",
  "task_count": 15,
  "rssi": -45, "wifi_connected": true,
  "ws_fps": 0, "mjpeg_fps": 0
}
```

实现: `main/system_info.c` (74 lines) — 独立模块，`system_info_register()` 模式。

### 2. HTTP Security Headers

所有 JSON 和 HTML 响应现在包含安全头部：

| Header | Value | 作用 |
|--------|-------|------|
| X-Content-Type-Options | nosniff | 防止 MIME 嗅探攻击 |
| X-Frame-Options | SAMEORIGIN | 防止点击劫持 |
| Cache-Control | no-store | 防止敏感数据缓存 |

实现方式：
- `http_set_security_headers()` — 内部 helper，统一设置
- `http_send_json()` — 已有函数增加安全头
- `http_send_html()` — 新增函数，HTML 响应 + 安全头
- `index_handler` / `stream_ws_page_handler` 改用 `http_send_html()`

## 技术笔记

- `esp_psram_get_size()` 需要 `esp_psram` 组件依赖，改用 `heap_caps_get_total_size(MALLOC_CAP_SPIRAM)` 避免额外依赖
- ESP-IDF httpd 不支持响应中间件，安全头需在每个 handler 中显式设置
- `http_send_html()` 集中化 HTML 响应的 content-type + security headers
- ESP32 chip revision 301 = v3.01 (ECO3)
