## Goals

Add system diagnostics capability, strengthen HTTP security, standardize integration tests.

## Completed Work

### 1. System Diagnostics API

Added `GET /api/system/info` endpoint returning device diagnostics:

```json
{
  "chip": { "model": "ESP32", "cores": 2, "revision": 301, "wifi": true, "bt": true },
  "idf_version": "v5.5.1",
  "memory": { "heap_free": 4169068, "heap_min": 4166160, "psram_free": 4037552, "psram_total": 4194304 },
  "uptime_s": 51, "uptime": "0d 00:00:51",
  "task_count": 15,
  "rssi": -45, "wifi_connected": true,
  "ws_fps": 0, "mjpeg_fps": 0
}
```

Implementation: `main/system_info.c` (74 lines) — standalone module, `system_info_register()` pattern.

### 2. HTTP Security Headers

All JSON and HTML responses now include security headers:

| Header | Value | Purpose |
|--------|-------|---------|
| X-Content-Type-Options | nosniff | Prevent MIME sniffing attacks |
| X-Frame-Options | SAMEORIGIN | Prevent clickjacking |
| Cache-Control | no-store | Prevent sensitive data caching |

Implementation:
- `http_set_security_headers()` — internal helper, unified setup
- `http_send_json()` — existing function enhanced with security headers
- `http_send_html()` — new function, HTML response + security headers
- `index_handler` / `stream_ws_page_handler` switched to `http_send_html()`

## Technical Notes

- `esp_psram_get_size()` requires `esp_psram` component dependency; switched to `heap_caps_get_total_size(MALLOC_CAP_SPIRAM)` to avoid extra dependency
- ESP-IDF httpd does not support response middleware; security headers must be set explicitly in each handler
- `http_send_html()` centralizes content-type + security headers for HTML responses
- ESP32 chip revision 301 = v3.01 (ECO3)
