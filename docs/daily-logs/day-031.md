# Day 31 — Stability & Robustness (Release Sprint 1/3)

**Date**: 2025-07-25
**Goal**: Harden firmware stability for stable release

## Completed Tasks

### 1. Task Watchdog Timer
- Subscribed main task to ESP-IDF task watchdog (`esp_task_wdt_add`)
- Enabled panic mode: auto-reboot on hang (`CONFIG_ESP_TASK_WDT_PANIC=y`)
- Increased timeout to 30s (was 10s) — suits camera streaming workload
- Watchdog reset in 1s main loop tick

### 2. Heap Integrity Monitoring
- Added `heap_caps_check_integrity_all()` check every 30s in main loop
- Low memory warning when internal free heap drops below 20KB
- `ESP_LOGE` on heap corruption detection
- Warning flag to avoid log spam

### 3. System Health API Endpoint
- Extended `/api/system/info` with `health` section:
  - `heap_ok` — heap integrity status
  - `memory_ok` — internal free heap above threshold
  - `wdt_timeout_s` — watchdog timeout value
  - `internal_free` — internal heap bytes free
- 3 new browser tests: Health heap/memory/watchdog

### 4. Error Handling Hardening
- `http_send_json()` — NULL check on `cJSON_PrintUnformatted()` (OOM protection)
- `status_handler()` — NULL check on `cJSON_CreateObject()` 
- `system_info_handler()` — NULL check on `cJSON_CreateObject()`
- Returns 500 on allocation failure instead of crash

### 5. App.js Connection Reliability Fix
- **Root cause**: MJPEG stream fetch on page load overloaded ESP32 HTTP connections
- **Fix**: Deferred MJPEG stream load by 500ms (`setTimeout`)
- Added null guard on canvas context initialization
- Fixed browser test timing: `wait_for_function` instead of `time.sleep`
