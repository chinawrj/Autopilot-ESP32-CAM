### 4. Dual-Server Architecture (Major Bug Fix)
**Issue**: MJPEG handler's `while(true)` loop blocks the ESP httpd task,
causing `/api/status` and other APIs to become unresponsive. This was a latent bug introduced on Day 3.

**Fix**: Split the MJPEG stream server into a separate httpd instance:
- Port 80: API + Pages + WebSocket (main server, no longer blocked)
- Port 81: MJPEG stream (separate server, stream_server.c)

**Verification**: Successfully ran concurrent curl `/api/status` while MJPEG stream was playing.
