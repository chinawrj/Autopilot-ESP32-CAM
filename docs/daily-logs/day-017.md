## 完成工作

### 1. /api/snapshot 端点
- 抓取单帧 JPEG 并返回 `image/jpeg`
- 支持 `Content-Disposition: inline; filename=snapshot.jpg`
- 测试: curl 返回 12KB JPEG, 640x480 有效图片

### 2. 统一入口页面 (index.html 重设计)
- Tab 切换: TCP Stream / WS Stream
- TCP 模式: `<img>` 标签从 port 81 加载 MJPEG
- WS 模式: Canvas + WebSocket 渲染，显示 Q/Resolution 控件
- Snapshot 按钮: 点击自动下载 JPEG (带时间戳文件名)
- LED Toggle 按钮
- 系统信息面板 (默认展开)
- OTA 面板 (默认折叠)

### 3. /api/status 增强
- 新增字段: `uptime` (秒), `rssi` (dBm), `wifi_connected` (bool)
- wifi_manager: 新增 `wifi_manager_get_rssi()` 函数

### 4. 双服务器架构 (重大 bug 修复)
**问题**: MJPEG handler 的 `while(true)` 循环阻塞 ESP httpd 任务，
导致 `/api/status` 等 API 无法响应。这是 Day 3 引入的潜在 bug。

**修复**: 将 MJPEG 流服务器拆分到独立的 httpd 实例:
- Port 80: API + Pages + WebSocket (主服务器, 不再被阻塞)
- Port 81: MJPEG 流 (独立服务器, stream_server.c)

**验证**: 在 MJPEG 流播放时并发 curl `/api/status` 成功。
