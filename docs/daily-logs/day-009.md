### Multi-Client Concurrency Test

Using `tools/multi_client_test.py` — 3 WS + 1 MJPEG, 60 seconds:

| Client | Frames | FPS | Data | Errors |
|--------|------|-----|--------|------|
| MJPEG | 614 | 10.2 | 6912 KB | 0 |
| WS0 | 264 | 4.4 | 2917 KB | 0 |
| WS1 | 265 | 4.4 | 2928 KB | 0 |
| WS2 | 0 | 0 | 0 KB | 0 (6 timeouts) |

- Final heap: 4111KB (min: 4064KB)
- **WS2 starvation**: ESP32 httpd default 7-socket limit, 3 WS + 1 MJPEG + status polling = 5 concurrent, LRU causes the 3rd WS client to be evicted
- **Conclusion**: 2 WS + 1 MJPEG is the optimal concurrent configuration. The 3rd WS client is unstable with 3 WS clients

### WiFi Infinite Reconnect (wifi_manager.c)
