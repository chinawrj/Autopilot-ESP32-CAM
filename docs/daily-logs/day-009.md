### 多客户端并发测试

使用 `tools/multi_client_test.py` — 3 WS + 1 MJPEG, 60 秒:

| 客户端 | 帧数 | FPS | 数据量 | 错误 |
|--------|------|-----|--------|------|
| MJPEG | 614 | 10.2 | 6912 KB | 0 |
| WS0 | 264 | 4.4 | 2917 KB | 0 |
| WS1 | 265 | 4.4 | 2928 KB | 0 |
| WS2 | 0 | 0 | 0 KB | 0 (6 timeouts) |

- Final heap: 4111KB (min: 4064KB)
- **WS2 starvation**: ESP32 httpd 默认 7 socket 上限，3 WS + 1 MJPEG + status polling = 5 并发，LRU 导致第 3 个 WS 客户端被挤出
- **结论**: 2 WS + 1 MJPEG 是最佳并发配置。3 个 WS 客户端时第 3 个不稳定

### WiFi 无限重连 (wifi_manager.c)

- 初始连接: 保持原有 MAX_RETRY 次数限制
- 连接成功后断线: 无限重试，带指数退避 (1s → 10s max)
- 使用 `s_initial_done` 标志区分初始/后续断线
- 串口输出重连日志: "WiFi lost, reconnect in Nms"
