## 发现的问题

### 问题 1: Port 81 被网络透明代理拦截 (严重)

**现象**:
- `curl http://192.168.1.171:81/stream/tcp` 返回 `Server: nginx`, 404

**根因**:
- 路由器/网络上存在 nginx 透明代理，拦截了 port 81 流量
- `192.168.1.171:81` 和 `192.168.1.1:81` 返回相同的 "Index of /" 页面
- ESP32 串口日志确认 stream server 确实在 port 81 上成功启动了

**修复**:
- 将 MJPEG stream server 端口从 81 改为 8081
- 更新 index.html 中的 stream URL
- 更新所有工具脚本和文档中的端口引用

**文件变更**:
- `main/stream_server.c` — port 81 → 8081
- `main/stream_server.h` — 注释更新
- `main/index.html` — stream URL 端口更新
- `main/main.c` — 注释更新
- `tools/quality_audit.py` — 新增质量审计脚本
