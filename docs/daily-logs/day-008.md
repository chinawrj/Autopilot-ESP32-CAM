## 完成内容

### WebSocket 控制消息

- **set_quality**: 客户端发送 `{"action":"set_quality","value":N}` (4-63)，immediately applies to camera sensor
- **set_resolution**: 客户端发送 `{"action":"set_resolution","value":"QVGA|VGA|SVGA|XGA"}`
- **get_status**: 触发即时心跳推送 (fps, clients, heap_free, heap_min)
- **架构决策**: 控制消息只修改设置+设置标志，不从 handler 发送响应 (避免与流推送任务的 socket 竞争)
  - 所有 WS 发送操作集中在 streaming task (Core 1)
  - 用 `volatile bool s_hb_now` 标志触发即时心跳

### 心跳机制

- 每 5 秒自动推送 JSON text frame 到所有连接的客户端
- 内容: `{"type":"heartbeat","fps":N,"clients":N,"heap_free":N,"heap_min":N}`
- get_status 命令触发即时心跳 (不等 5 秒)

### Heap 监控

- `/api/status` 新增字段: `heap_free`, `heap_min`
- WS 心跳包含相同的 heap 数据
- 当前 heap: ~4125KB free, ~4107KB minimum

### 前端更新 (stream_udp.html)

- HUD 新增 HEAP 显示 (从心跳更新)
- Quality 下拉菜单: Q:10 Best / Q:12 / Q:20 / Q:30 / Q:50 Fast
- Resolution 下拉菜单: QVGA / VGA / SVGA / XGA
- Status bar 显示 clients + heap 详情

### Bug 修复

- **WS socket send race condition**: handler 尝试 `httpd_ws_send_frame()` 同时 streaming task 调用 `httpd_ws_send_frame_async()` → errno 119 + 连接断开
- 修复: 控制消息 handler 不再发送响应，改为设置 `s_hb_now` 标志由 streaming task 统一发送
