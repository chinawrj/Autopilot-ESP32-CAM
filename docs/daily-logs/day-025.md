# Day 25 — GitHub Actions CI/CD Pipeline

**日期**: $(date +%Y-%m-%d)
**固件版本**: v1.3.0 (无版本变更)

## 目标

为项目添加 GitHub Actions CI/CD pipeline，每次 push/PR 自动验证：
1. Host-based 单元测试 (20 tests)
2. ESP-IDF 固件编译 (0 warnings)

## 完成内容

### 1. CI Workflow (`.github/workflows/ci.yml`)

**两个独立 Job:**

| Job | 运行环境 | 内容 | 预计时间 |
|-----|---------|------|---------|
| Unit Tests | ubuntu-latest | 下载 Unity → cmake → ctest | ~1 min |
| ESP-IDF Build | espressif/idf:v5.5.1 | idf.py build + 产物上传 | ~5 min |

**触发条件:** push to main, PR to main

### 2. Test CMakeLists.txt 改进

使 UNITY_DIR 可通过以下方式配置（按优先级）：
1. CMake 变量 `-DUNITY_DIR=...`
2. 环境变量 `UNITY_DIR`
3. `$IDF_PATH/components/unity/unity/src`
4. `$HOME/esp/esp-idf/components/unity/unity/src` (fallback)

### 3. README CI Badge

双语 README 添加 CI status badge，链接到 Actions 页面。
