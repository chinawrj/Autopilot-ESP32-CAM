## Today's Goals

Implement HTTP OTA wireless firmware update feature (v1.1.0 roadmap Day 16)

## Completed Work

### 1. Partition Table Changed to Dual OTA Layout

| Partition | Offset | Size | Description |
|-----------|--------|------|-------------|
| nvs | 0x9000 | 16KB | NVS storage |
| otadata | 0xd000 | 8KB | OTA boot selection |
| phy_init | 0xf000 | 4KB | PHY calibration |
| ota_0 | 0x10000 | 3MB | Application partition A |
| ota_1 | 0x310000 | 3MB | Application partition B |
| storage | 0x610000 | ~2MB | SPIFFS |

Firmware size: ~1.15MB, 63% partition space remaining.

### 2. OTA Module (ota_update.c/h, 119 lines)

- Background task downloads firmware without blocking the HTTP server
- Real-time progress percentage tracking (21%→44%→58%→69%→85%→100%)
- State machine: idle → downloading → success/error
- Version number read from `esp_app_desc_t`

### 3. REST API

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/ota` | POST | `{"url":"http://..."}` triggers update |
| `/api/ota/status` | GET | Progress/status/version query |
| `/api/status` | GET | Added `version` field |

### 4. Frontend OTA Panel

Added collapsible "System & OTA Update" panel to index.html:
- Firmware version display
- URL input field + Update button
- Progress bar + status text
- 500ms progress polling

### 5. Bug Fix: Camera PWDN Power-Up Sequence

**Issue**: I2C probe timeout after OTA soft reboot (camera probe failed)
**Root cause**: `esp_restart()` does not reset GPIOs, leaving camera module in undefined state
**Fix**: Manually toggle PWDN pin before `esp_camera_init` (HIGH→100ms→LOW→100ms)
